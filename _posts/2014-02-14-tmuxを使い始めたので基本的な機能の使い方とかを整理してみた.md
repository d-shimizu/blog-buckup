<body>
<h4>はじめに</h4>
<p>tmuxを真剣に使い始めて1ヶ月程度ですが、tmuxについて基本的な機能の使い方をまとめてみようと思います。<br>類似の記事は既にたくさんありますが、とりあえず最低限覚えておくと良さそうな機能について整理してみます。 </p>
<a name="more"></a> <h4>tmuxとは</h4>
<p>tmuxは端末多重化ソフトウェアです。"てぃーまっくす"と読まれます。<br><a class="keyword" href="http://d.hatena.ne.jp/keyword/Unix">Unix</a>/<a class="keyword" href="http://d.hatena.ne.jp/keyword/Linux">Linux</a>系OSを利用されている方であれば端末を利用した作業はほぼ必須かと思いますが、そのような場合にtmuxを利用できれば、1つのターミナルで操作しているシェルから実行したtmux上で複数の仮想端末を操作できるため、ターミナルを複数開くことなく複数のサーバへログインしたり、それぞれの仮想端末で別々のプログラムを実行できるようになるため、より効率的に作業が行えます。 </p>
<p>具体的には以下のような形です。 </p>
<h6>tmuxがない場合</h6>
<ul>  <li>
<a class="keyword" href="http://d.hatena.ne.jp/keyword/ssh">ssh</a>接続するたびに端末が開かれる</li>  <ul>    <li>画面のカオス化</li>  </ul>  <li>回線が切れる</li>  <ul>    <li>実行中の処理が途切れてゲームオーバー</li>  </ul>  <li>
<a class="keyword" href="http://d.hatena.ne.jp/keyword/ssh">ssh</a>クライアント実行中の端末が停止</li>  <ul>    <li>
<a class="keyword" href="http://d.hatena.ne.jp/keyword/Windows">Windows</a>だとブルースクリーンになって実行中の処理が (ry</li>  </ul>
</ul> <h6>tmuxを使うことによるメリット</h6>
<ul>  <li>1つの端末で複数の擬似端末を起動可能</li>  <ul>    <li>複数の端末を立ち上げずに、tmux上の擬似端末を切り替えてオペレーション可能</li>  </ul>  <li>起動した仮想端末を画面分割して使用可能</li>  <ul>    <li>他のファイルを参照したりログ出力を参照しながらオペレーション可能</li>  </ul>  <li>起動した仮想端末上でコピペが可能</li>  <ul>    <li>マウスを使わず、tmux内でキーボードのみでのコピペが可能</li>  </ul>  <li>起動した仮想端末のデタッチ(切り離し)/アタッチ(接続)が可能</li>  <ul>    <li>tmux実行端末とのネットワークが切れても問題なく、異なる環境から同じtmuxセッションへ接続可能</li>  </ul>
</ul> <p>こんなに散らばっていた複数のターミナル画面ですが… </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/-VokEmNYxZ5Q/UvnWiVCvnNI/AAAAAAAAAbM/YGcwPOa7oew/s1600/%25E3%2582%25B9%25E3%2582%25AF%25E3%2583%25AA%25E3%2583%25BC%25E3%2583%25B3%25E3%2582%25B7%25E3%2583%25A7%25E3%2583%2583%25E3%2583%2588+2014-02-11+16.11.48.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="https://cdn-ak.f.st-hatena.com/images/fotolife/d/dshimizu/20220322/20220322113739.png"></a></div>
<div class="separator" style="clear: both; text-align: center;"></div>
<p>こんな風に1つのターミナル画面にまとめることができます。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-JeZJlyhY_Ek/UvnWkb_JkqI/AAAAAAAAAbU/x8aoKIo8rus/s1600/%25E3%2582%25B9%25E3%2582%25AF%25E3%2583%25AA%25E3%2583%25BC%25E3%2583%25B3%25E3%2582%25B7%25E3%2583%25A7%25E3%2583%2583%25E3%2583%2588+2014-02-11+16.30.26.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-JeZJlyhY_Ek/UvnWkb_JkqI/AAAAAAAAAbU/x8aoKIo8rus/s640/%25E3%2582%25B9%25E3%2582%25AF%25E3%2583%25AA%25E3%2583%25BC%25E3%2583%25B3%25E3%2582%25B7%25E3%2583%25A7%25E3%2583%2583%25E3%2583%2588+2014-02-11+16.30.26.png"></a></div> <h5>tmux利用のイメージ図</h5>
<p>利用イメージ図としては以下のような感じです。<br><a class="keyword" href="http://d.hatena.ne.jp/keyword/ssh">ssh</a>先の踏み台サーバ上でtmuxを起動して各サーバ群に<a class="keyword" href="http://d.hatena.ne.jp/keyword/ssh">ssh</a>接続します。私はウィンドウ毎に接続先サーバを分ける等して使ってます。tmuxセッションが残っていれば、自端末と踏み台サーバ間の通信は切れても問題ありません。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://3.bp.blogspot.com/-e4Sylm_HwDM/UvyWnuU63wI/AAAAAAAAAe0/RHA87XGTwQ8/s1600/tmux.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://3.bp.blogspot.com/-e4Sylm_HwDM/UvyWnuU63wI/AAAAAAAAAe0/RHA87XGTwQ8/s640/tmux.png"></a></div> <h5>tmuxの構成要素</h5>
<p>ここで、tmuxの構成要素について簡単に記載します。 </p> <h6>設定ファイル</h6>
<p><code>~/.tmux.conf</code> に設定を記述します。新規セッション作成時に読み込まれます。<br>最初は存在しませんので、デフォルトの設定をカスタマイズするような場合に自分で作成する必要があります。<br>CentOS6であればtmuxインストール後に<code>/usr/share/doc/tmux-1.6/examples/</code>以下に設定ファイルのサンプルが設置されますので参考にすると良いかもしれません。 </p> <h6>tmuxのプロセス構成</h6>
<p>tmuxはサーバ/クライアント構成となります。 </p>
<ul>  <li>tmuxサーバ(セッション)</li>  <ul>    <li>tmux起動時に生成される、tmuxセッションを管理しているプロセス</li>  </ul>  <li>tmuxクライアント(pty/仮想端末)</li>  <ul>    <li>tmuxセッションへ接続(アタッチ)しているプロセス(pty)</li>  </ul>
</ul> <h6>画面構成要素</h6>
<ul>  <li>セッション(Session)</li>  <ul>    <li>tmuxに管理される仮想端末全体を指す</li>    <li>各セッションは1つ以上のウィンドウを持つ</li>  </ul>  <li>ウィンドウ(Window)</li>  <ul>    <li>tmuxセッション内に開かれている1つの仮想端末の画面全体を指す</li>    <li>ウィンドウは1つ以上のペインを持つ</li>  </ul>  <li>ペイン(Pane)</li>  <ul>      <li>分割されたウィンドウの単位を指す</li>    <li>ペインそれぞれが独立した擬似端末となる</li>  </ul>
</ul> <p>実際の画面を元に示すと、青い枠がペイン、緑の枠がウィンドウ、青と緑を合わせてセッションを指します。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-O1xarvqTZKo/Uvn4SqGHGYI/AAAAAAAAAbg/-Sub5rcABbo/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-11+16.30.26.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-O1xarvqTZKo/Uvn4SqGHGYI/AAAAAAAAAbg/-Sub5rcABbo/s640/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-11+16.30.26.png"></a></div> <h4>tmuxを使ってみる</h4>
<p>文章での説明だとよくわからない部分も多いかもしれませんので、実際にtmuxを使ってみます。 </p>
<h5>tmuxのインストール</h5>
<p>tmuxをインストールしてみます。<br>最新バージョン1.8の<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%BD%A1%BC%A5%B9%A5%B3%A1%BC%A5%C9">ソースコード</a>をダウンロードしてインストールすることもできますが、各OSのパッケージ管理ツールを利用してインストールするのが簡単です。 </p>
<p><a class="keyword" href="http://d.hatena.ne.jp/keyword/RHEL">RHEL</a>系<a class="keyword" href="http://d.hatena.ne.jp/keyword/Linux">Linux</a>であればEPEL<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%EA%A5%DD%A5%B8%A5%C8%A5%EA">リポジトリ</a>にパッケージが存在しますので<a class="keyword" href="http://d.hatena.ne.jp/keyword/yum">yum</a>でパッケージをインストールできます。 </p>
<pre class="terminal">  # yum install http://dl.fedoraproject.org/pub/epel/6/x86_64/tmux-1.6-3.el6.x86_64.rpm  
</pre>
<p><a class="keyword" href="http://d.hatena.ne.jp/keyword/OS%20X">OS X</a>であればhomebrewからインストールできます。 </p>
<pre class="terminal">  # brew install tmux  </pre> <p>以下では<a class="keyword" href="http://d.hatena.ne.jp/keyword/Mac%20OS%20X">Mac OS X</a> 10.7のiTermから<a class="keyword" href="http://d.hatena.ne.jp/keyword/CentOS">CentOS</a> 6.5のサーバへログインし、<a class="keyword" href="http://d.hatena.ne.jp/keyword/CentOS">CentOS</a> 6.5で起動したtmux 1.6で動作確認した内容を記載します。 </p> <h5>tmuxの基本操作</h5>
<p>起動中(アタッチしている)のtmuxを操作するには、多くの場合tmuxの制御を行うことを示すための<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%D7%A5%EC%A5%D5%A5%A3%A5%C3%A5%AF%A5%B9">プレフィックス</a>キー<code>Ctrl-b</code>(デフォルト設定値)を入力し、それに続いて実行したい操作に対応するコマンドキーを入力する形でtmuxの内部コマンドを実行することになります。<br>どのキーがどの操作に対応しているかは <code>Ctrl-b</code> + <code>?</code> を入力することで表示できます。 </p>
<h6>tmuxの起動</h6>
<p>tmuxを起動してセッションを開始します。起動には以下のコマンドを実行します。 </p>
<pre class="terminal">  % tmux  
</pre>
<p>または、 </p>
<pre class="terminal">  % tmux new-session  
</pre>
<p>これでtmuxセッションが作成され、セッションにアタッチした状態になります。以下のような画面になります。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/-n5hrKyAqdyY/UvpRa4WOxGI/AAAAAAAAAcQ/Cbx_jV4YLTg/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-12+01.33.28.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="https://cdn-ak.f.st-hatena.com/images/fotolife/d/dshimizu/20220322/20220322113803.png"></a></div> <h6>セッションの確認</h6>
<p>ここで、起動しているtmuxのセッションについて確認してみます。前述の通り、tmuxにはtmux全体を管理しているセッションと、セッションに接続しているクライアントの2種類があります。 </p>
<p>現在起動中のセッションを確認するには<code>tmux list-sessions</code> または <code>tmux ls</code>を実行します。左側の数字がセッション番号になります。以下の例ではセッション番号が0となっていることが分かります。 </p>
<pre class="terminal">  % tmux list-sessions 0: 1 windows (created Tue Feb 11 21:43:00 2014) [193x36] (attached)  </pre> <p>現在起動中のセッションに接続されているクライアントを確認するには<code>tmux list-client</code> または <code>tmux lsc</code> を実行します。 </p>
<pre class="terminal">  % tmux list-client /dev/pts/1: 0 [37x193 xterm] (utf8)  </pre> <h6>セッションのデタッチ</h6>
<p>セッションをデタッチするには<code>Ctrl-b</code> + <code>d</code> または <code>Ctrl-b</code> + <code>: detach-client</code>を実行します。なお、回線が切断された時には自動でデタッチされます。<br>仮想端末が終了し、以下の出力とともに元の端末に戻ります。 </p>
<pre class="terminal">  [detached] %  
</pre>
<p>デタッチによって、tmuxセッションで実行中の処理を継続させたままクライアントを終了できます。再度そのセッションへアタッチすることで作業の続きを行うことができます。 </p> <h6>セッションのアタッチ</h6>
<p>アタッチするにはtmuxを起動するには <code>tmux attach</code> を実行します。attachの文字を全て入力せずとも <code>tmux a</code> 等でもいけます。<br>複数のtmuxセッションを起動している場合は、<code>-t [target-session]</code> オプションで接続したいセッションを指定することができます。セッション番号は前述の <code>tmux list-sessions</code> で参照することができます。<br>なお，ひとつのセッションに複数のクライアントをアタッチすることも可能です。<code>-d</code> を指定すると、そのセッションにアタッチしている他のクライアントはデタッチされます。 </p>
<pre class="terminal">  % tmux attach -t 0  </pre> <h6>セッションの削除</h6>
<p><code>tmux kill-session</code> で現在起動中のセッションを削除できます。<code>-t [target-session]</code> オプションを指定することで削除したいセッションを指定することができますが、指定しない場合は直近でアタッチしていたセッションが削除されます。 </p>
<pre class="terminal">  % tmux kill-session  
</pre>
<p>全てのセッションを削除するには <code>tmux kill-server</code> を実行します。 </p>
<pre class="terminal">  % tmux kill-server  </pre> <h5>ウィンドウの操作</h5>
<h6>新規ウィンドウ作成</h6>
<p>セッションにアタッチした状態で<code>Ctrl-b</code> + <code>c</code> または <code>:new-window</code> を入力することで新規ウィンドウを作成できます。<br></p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/-vHnIsBI9Pf4/UvpSLrvxS8I/AAAAAAAAAco/CuUITWtCi74/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-12+1.32.42.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="https://cdn-ak.f.st-hatena.com/images/fotolife/d/dshimizu/20220322/20220322113823.png"></a></div>
<p>画面下に<code>0:zsh-</code> に続いて <code>1:zsh*</code>と表示されますが、これがそれぞれウィンドウとなります。現在選択されているウィンドウには <code>*</code> と表示されており、それ以外のウィンドウには <code>-</code> と表示されます。ウィンドウ名は <code>Ctrl-b</code> + <code>:rename-window [window name]</code> で指定できます。 </p> <h6>ウィンドウの切り替え</h6>
<p><code>Ctrl-b</code> + <code>[ウィンドウ番号]</code>でウィンドウを切り替えることができます。<br>また、<code>Ctrl-b</code> + <code>n</code> で上の番号のウィンドウへ、<code>Ctrl-b</code> + <code>p</code> で下の番号のウィンドウへ切り替えることができます。 上記の画面上で<code>Ctrl-b</code> + <code>0</code>と入力すると、ウィンドウ番号0の画面が表示されます。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-WJgXEfp09SQ/UvuWlGi4nUI/AAAAAAAAAc4/AQuU2vl-o-A/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+0.42.44.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-WJgXEfp09SQ/UvuWlGi4nUI/AAAAAAAAAc4/AQuU2vl-o-A/s640/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+0.42.44.png"></a></div> <h6>ウィンドウ一覧の表示</h6>
<p><code>Ctrl-b</code> + <code>w</code> で現在作成されているウィンドウの一覧が表示されます。<br><code>↑</code> で <code>↓</code> でウィンドウを選択して <code>Enter</code> キーで対象ウィンドウを表示させることもできます。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/-KUfB722B6kM/UvuY_V7ZnDI/AAAAAAAAAdE/Ye34E873Rks/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+0.52.42.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="https://cdn-ak.f.st-hatena.com/images/fotolife/d/dshimizu/20220322/20220322113707.png"></a></div> <h6>ウィンドウの削除</h6>
<p><code>Ctrl-b</code> + <code>&amp;</code> と入力することで作成したウィンドウを削除できます。<code>Confirm 'kill-window'? (y/n)</code> というメッセージが出力されますので、<code>y</code> と入力するとウィンドウが削除されます。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://4.bp.blogspot.com/-bmFJCV6ujOA/Uvzb5n6nzzI/AAAAAAAAAfA/HD_dF1rIDKE/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+23.49.07.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-bmFJCV6ujOA/Uvzb5n6nzzI/AAAAAAAAAfA/HD_dF1rIDKE/s640/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+23.49.07.png"></a></div> <h5>ペインの操作</h5>
<h6>水平分割</h6>
<p><code>Ctrl-b</code> + <code>"</code> でウィンドウを水平に分割できます。これで以下のような画面構成になり、2つのペインが作成されます。 </p>
<div class="separator" style="clear: both; text-align: center;"><a href="http://4.bp.blogspot.com/-84TSYPLo4OQ/UvuchRsW_9I/AAAAAAAAAdQ/E_qHcPmqcFE/s1600/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+1.07.39.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-84TSYPLo4OQ/UvuchRsW_9I/AAAAAAAAAdQ/E_qHcPmqcFE/s640/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88+2014-02-13+1.07.39.png"></a></div>
<h6>垂直分割</h6>
<p><strike> <code>Ctrl-b</code> + <code>$</code> でウィンドウを垂直に分割できます。</strike>(※2014/10/21コメント指摘事項修正) <code>Ctrl-b</code> + <code>%</code> でウィンドウを垂直に分割できます。これで以下のような画面構成になり、3つ目のペインが作成されます。 </p>

<p>ペイン間の移動は <code>Ctrl-b</code> + <code>o</code> で行います。<br>また、<code>Ctrl-b</code> + <code>q</code> または <code>Ctrl-b</code> + <code>:display-panes</code> で以下のようにペインのインジケータ(番号)を表示させることができ、<code>Ctrl-b</code> + <code>q</code> + <code>インジケータ番号</code> で指定したペインへ移動することもできます。 <br>インジケータが表示される時間はデフォルトだと少々短いですが、<code>Ctrl-b</code> + <code>:display-panes-time [time]</code> で変更できます。単位はミリ秒なので、表示時間を1秒にしたい場合は1000と指定します。  </p>
<p>以下ではインジケータ番号2のペインが選択されている状態ですが、<code>Ctrl-b</code> + <code>q</code> でインジケータ番号が表示されている間に <code>0</code> を入力します。 </p>

<p><code>Ctrl-b</code> + <code>x</code> でペイン分割を解除できます。 </p>
<p>インジケータ番号0のペインが選択されている状態で<code>Ctrl-b</code> + <code>x</code> を入力すると、画面下のステータスバーに <code>kill-pane 0? (y/n)</code> と表示されます。 </p>
